### 五、教学过程

#### 第一环节：导入（10分钟）

**教学内容**
- 引入问题：如何用数据讲故事？
  - 数据分析的最终目的是支持决策
  - 好的报告能让决策者快速理解分析结果

- 报告的作用
  - 传达分析结果
  - 支持业务决策
  - 推动行动落地

**教学活动**
- 教师提问：你看过好的数据分析报告吗？
- 学生思考：好的报告有什么特点？

---

#### 第二环节：数据可视化原则（30分钟）

**教学内容**
- 可视化的目的
  - 让数据更容易理解
  - 发现数据中的模式和规律
  - 支持决策和沟通

- 可视化的基本原则

| 原则 | 含义 | 示例 |
| :--- | :--- | :--- |
| **准确性** | 准确传达数据信息 | 不要扭曲数据 |
| **简洁性** | 去除不必要的元素 | 避免过度装饰 |
| **可读性** | 易于理解 | 使用清晰的标签 |
| **一致性** | 保持风格统一 | 使用统一的颜色 |

- 常见图表类型与适用场景

| 图表类型 | 适用场景 | 示例 |
| :--- | :--- | :--- |
| **折线图** | 展示趋势 | 价格走势、销量变化 |
| **柱状图** | 展示对比 | 不同平台的价格对比 |
| **饼图** | 展示占比 | 市场份额 |
| **散点图** | 展示相关性 | 价格与销量的关系 |
| **热力图** | 展示分布 | 用户活跃度分布 |

- 常见可视化错误
  - 使用3D图表扭曲数据
  - 使用不当的颜色对比
  - 缺少必要的标签和说明
  - 数据墨水比过低

**教学活动**
- 教师讲解可视化原则
- 学生识别可视化错误

---

#### 第三环节：报告撰写规范（30分钟）

**教学内容**
- 报告的基本结构

| 部分 | 内容 | 要点 |
| :--- | :--- | :--- |
| **摘要** | 核心发现 | 简洁、突出重点 |
| **背景** | 分析背景和目标 | 说明为什么做这个分析 |
| **方法** | 数据来源和分析方法 | 说明怎么做的 |
| **结果** | 分析结果 | 用图表展示 |
| **结论** | 核心发现和建议 | 回答业务问题 |
| **附录** | 详细数据和代码 | 供参考 |

- 各部分的写作要点
  - 摘要：1-2句话总结核心发现
  - 背景：说明业务问题和分析目标
  - 方法：说明数据来源、分析方法、工具
  - 结果：用图表展示，配以文字说明
  - 结论：回答业务问题，提出建议
  - 附录：详细数据、代码、参考资料

- 报告撰写的常见问题
  - 缺乏明确的分析目标
  - 数据展示混乱
  - 结论不清晰
  - 缺乏可操作的建议

**教学活动**
- 教师讲解报告撰写规范
- 学生理解报告结构

---

#### 第四环节：案例分析（20分钟）

**教学内容**
- 案例：China's Prices Project的报告撰写
  - 分析目标：计算每日线上价格指数
  - 数据来源：电商平台价格数据
  - 分析方法：描述性统计、时间序列分析
  - 报告结构：
    - 摘要：价格指数的核心发现
    - 背景：传统CPI的局限性
    - 方法：数据采集和分析方法
    - 结果：价格指数的走势和分布
    - 结论：对市场趋势的判断和建议

- 优秀报告案例分析
  - 结构清晰
  - 图表规范
  - 结论明确
  - 建议可操作

**教学活动**
- 教师演示报告撰写过程
- 学生理解报告撰写要点

---

#### 第五环节：课堂研讨（15分钟）

**研讨议题一：报告撰写练习**
- 讨论题：请根据China's Prices Project的数据，撰写一份简要的分析报告，包括：
  1. 摘要
  2. 背景
  3. 方法
  4. 结果
  5. 结论
- 目的：练习报告撰写

**研讨议题二：报告评审与反馈**
- 讨论题：各组汇报报告后，其他组提出问题和建议：
  1. 报告结构是否清晰？
  2. 图表是否规范？
  3. 结论是否明确？
  4. 建议是否可操作？
- 目的：培养报告评审能力

**教学活动**
- 学生分组撰写报告
- 各组汇报报告
- 教师点评与总结
