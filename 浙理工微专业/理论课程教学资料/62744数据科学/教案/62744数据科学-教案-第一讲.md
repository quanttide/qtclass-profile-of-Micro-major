### 五、教学过程

#### 第一环节：导入（20分钟）

**教学内容**
- 引入问题：什么是数据科学？
  - 数据科学是用数据解决实际问题的学科
  - 数据科学 = 统计学 + 计算机科学 + 业务知识

- 数据科学的应用场景
  - 电商推荐：猜你喜欢
  - 金融风控：识别欺诈
  - 医疗诊断：辅助诊断
  - 智能交通：路线规划

**教学活动**
- 教师提问：你接触过哪些数据科学应用？
- 学生分享自己的经历

---

#### 第二环节：数据科学概述（40分钟）

**教学内容**
- 数据科学的定义
  - 数据科学是从数据中提取知识和洞察的学科
  - 数据科学是用数据驱动决策的方法论

- 数据科学与相关领域的关系

| 领域 | 核心任务 | 技能要求 |
| :--- | :--- | :--- |
| **数据科学** | 从数据中提取知识和洞察 | 统计学、机器学习、业务理解 |
| **数据分析** | 描述和解释数据 | SQL、Excel、可视化 |
| **数据工程** | 构建数据基础设施 | 分布式系统、ETL、数据管道 |
| **机器学习** | 构建预测模型 | 算法、编程、数学 |

- 数据科学家的角色与技能要求
  - 技术能力：编程、统计、机器学习
  - 业务能力：理解业务问题、沟通表达
  - 软技能：好奇心、批判性思维

**教学活动**
- 教师讲解数据科学的定义与范畴
- 学生理解数据科学与相关领域的区别

---

#### 第三环节：业务场景分析（40分钟）

**教学内容**
- 案例：China's Prices Project
  - 业务背景：经济学课题组需要计算每日线上价格指数
  - 业务问题：传统CPI依赖人工调研，时效性差、颗粒度粗
  - 数据科学解决方案：利用电商数据构建高频价格指数

- 业务问题转化为数据问题的方法
  1. 明确业务目标：要解决什么问题？
  2. 定义数据需求：需要什么数据？
  3. 确定分析方法：用什么方法分析？
  4. 设计评估指标：如何评估效果？

- 问题定义的SMART原则
  - Specific（具体）：问题要明确
  - Measurable（可衡量）：结果要可量化
  - Achievable（可实现）：目标要现实
  - Relevant（相关）：与业务相关
  - Time-bound（有时限）：有明确的时间要求

**教学活动**
- 教师讲解业务场景分析方法
- 学生用China's Prices Project练习问题定义

---

#### 第四环节：建模基础（30分钟）

**教学内容**
- 什么是模型
  - 模型是对现实世界的简化表示
  - 模型是用数学公式描述数据中的规律

- 建模的基本流程

```
问题定义 → 数据收集 → 数据预处理 → 特征工程 → 模型选择 → 模型训练 → 模型评估 → 模型部署
```

- 常见的建模任务

| 任务类型 | 定义 | 示例 |
| :--- | :--- | :--- |
| **分类** | 预测离散标签 | 垃圾邮件识别、用户画像 |
| **回归** | 预测连续值 | 价格预测、销量预测 |
| **聚类** | 发现数据分组 | 用户分群、异常检测 |
| **推荐** | 推荐相关内容 | 商品推荐、内容推荐 |

- 模型评估的基本概念
  - 准确率（Accuracy）：预测正确的比例
  - 精确率（Precision）：预测为正的样本中真正为正的比例
  - 召回率（Recall）：真正为正的样本中被预测为正的比例
  - F1值：精确率和召回率的调和平均

**教学活动**
- 教师讲解建模的基本概念
- 学生理解建模流程

---

#### 第五环节：课堂研讨（20分钟）

**研讨议题一：场景识别与问题定义**
- 讨论题：请选择一个你熟悉的业务场景（如电商、金融、医疗等），分析：
  1. 这个场景中有什么业务问题？
  2. 这些问题如何转化为数据问题？
  3. 需要用什么建模方法？
- 目的：练习业务场景分析与问题定义

**研讨议题二：建模流程设计**
- 讨论题：针对China's Prices Project，请设计一个建模流程：
  1. 需要什么数据？
  2. 如何进行数据预处理？
  3. 选择什么建模方法？
  4. 如何评估模型效果？
- 目的：练习建模流程设计

**教学活动**
- 学生分组讨论
- 各组汇报讨论结果
- 教师点评与总结

---

- 强调了业务场景分析与问题定义的方法

- 通过China's Prices Project引入了本课程的贯穿案例
