### 五、教学过程

#### 第一环节：导入（15分钟）

**教学内容**
- 引入问题：现代数据技术栈是什么？
  - 企业如何存储海量数据？
  - 企业如何快速处理数据？
  - 企业如何管理数据？

- 现代数据技术栈的五个核心模块
  1. 现代数据仓库（ClickHouse、Snowflake）
  2. 数据湖与湖仓一体（Databricks）
  3. 计算引擎与批流一体（Spark、Flink）
  4. 元数据管理（DataHub、OpenMetadata）
  5. 现代架构思想（Data Mesh、Data Fabric）

**教学活动**
- 教师介绍五个模块
- 学生了解本节课的学习任务

---

#### 第二环节：小组研究与汇报（120分钟）

**教学内容**

**小组1：现代数据仓库（ClickHouse & Snowflake）**
- 核心定义：数据仓库是面向分析的"超级档案柜"
- ClickHouse：开源列式数据库，擅长秒级完成亿级数据的聚合分析
- Snowflake：云原生数据仓库服务，存算分离，完全免运维
- 一句话总结：解决了"数据存不下"和"查得慢"的问题

**小组2：数据湖与湖仓一体（Databricks）**
- 核心定义：从杂乱原始的"数据沼泽"进化为整洁规范的"数据基地"
- 数据湖：存储成本低，什么都能往里扔，但难以直接分析
- 湖仓一体：结合数据湖的灵活性和数据仓库的管理能力
- 一句话总结：解决了"数据格式杂乱"和"管理困难"的问题

**小组3：计算引擎与批流一体（Spark & Flink）**
- 核心定义：大数据的"发动机"，负责把数据跑起来
- Spark：内存计算的王者，擅长处理批量数据
- Flink：流计算的霸主，擅长处理实时数据
- 批流一体：用一套代码同时处理历史和实时数据
- 一句话总结：解决了"数据怎么算"和"算得快不快"的问题

**小组4：元数据管理（DataHub & OpenMetadata）**
- 核心定义：数据的"户口本"和"导航地图"
- 痛点：公司有上万张表，没人知道哪张表是干嘛的
- DataHub/OpenMetadata：数据目录，自动扫描数据库，记录表的含义、字段解释、负责人
- 一句话总结：解决了"数据在哪"、"是什么"以及"谁在用"的问题

**小组5：现代架构思想（Data Mesh & Data Fabric）**
- 核心定义：组织如何规模化治理数据的哲学
- Data Mesh：去中心化，每个业务部门自己负责自己的数据产品
- Data Fabric：连接与自动化，利用AI和知识图谱自动编织数据
- 一句话总结：解决了"数据治理如何规模化"和"如何智能化连接"的问题

**教学活动**
- 学生分组研究（每组研究一个模块）
- 各组汇报研究成果（每组15分钟）
- 其他组提问与讨论

---

#### 第三环节：技术架构全景图（20分钟）

**教学内容**
- 现代数据技术栈架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    现代架构思想                              │
│              Data Mesh / Data Fabric                        │
├─────────────────────────────────────────────────────────────┤
│                    元数据管理                                │
│              DataHub / OpenMetadata                         │
├─────────────────────────────────────────────────────────────┤
│                    计算引擎                                  │
│              Spark (批处理) / Flink (流处理)                 │
├─────────────────────────────────────────────────────────────┤
│                    存储层                                    │
│     数据仓库 (ClickHouse/Snowflake)                         │
│     数据湖 (S3/HDFS) + 表格式 (Iceberg/Delta Lake)          │
│     湖仓一体 (Databricks)                                   │
└─────────────────────────────────────────────────────────────┘
```

- 各模块之间的关系
  - 存储层是地基，存放数据
  - 计算引擎是发动机，处理数据
  - 元数据管理是导航图，管理数据
  - 现代架构思想是哲学，指导数据治理

**教学活动**
- 教师展示架构图
- 学生理解各模块的定位与关系

---

#### 第四环节：课堂研讨（15分钟）

**研讨议题：技术选型**
- 讨论题：假设你所在的公司需要搭建一个大数据平台，主要需求是：
  1. 每天处理100GB的用户行为日志
  2. 需要支持实时监控大屏
  3. 团队规模较小，希望降低运维成本
- 请从存储、计算、元数据管理三个维度，推荐合适的技术方案。
- 目的：培养技术选型能力

**教学活动**
- 学生分组讨论
- 各组汇报技术选型方案
- 教师点评与总结

---

#### 第五环节：课堂小结（10分钟）

**小结内容**
- 本讲介绍了现代数据技术栈的五个核心模块
- 数据仓库、数据湖、湖仓一体是存储层的三种方案
- Spark和Flink是计算引擎的两大主流
- 元数据管理解决了"数据在哪"的问题
- Data Mesh和Data Fabric是现代数据治理的架构思想
- 理解这些技术组件，是后续学习数据采集和计算的基础
