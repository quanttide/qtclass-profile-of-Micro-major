## 3.2 案例导入：电商价格指数计算

### 3.2.1 CPI是什么

CPI，你可以把它理解为"国家给大家算的一笔生活账"。想象一下，国家统计局每个月都会派调查员去超市、商场、菜市场，记录一篮子日常生活必需的东西的价格，比如肉蛋菜、衣服、房租、公交车票、电影票等等。CPI就是把这"一篮子东西"的总价格，和上个月、去年同个月份进行比较，算出一个百分比。

它有两大主要作用：
1. 它是经济的"体温计"，国家看着它来判断经济是"发烧"（通胀太高）还是"过冷"（通缩）
2. 它是很多钱的"调节器"，比如一些退休金、最低工资的调整，经常会参考CPI的涨幅

### 3.2.2 项目背景

消费者价格指数（CPI）是衡量通胀水平的重要宏观经济指标，但传统CPI依赖人工调研与月度统计更新，存在时效性差、颗粒度粗、覆盖不全等固有缺陷。随着电商平台数据的大规模生成，构建高频（按日）价格指数成为可能。

## 3.3 数据采集方式

### 3.3.1 API调用

**定义**：通过官方接口获取数据

**适用场景**：有开放API的平台

**优缺点**：
- 优点：合规、稳定、数据结构化
- 缺点：数据量有限、可能有调用限制

**示例**：淘宝开放平台API

### 3.3.2 Web爬虫

**定义**：模拟浏览器访问网页

**适用场景**：无API的网站

**优缺点**：
- 优点：灵活、可获取任意网页数据
- 缺点：可能违反网站协议、需要处理反爬

**示例**：爬取商品价格数据

### 3.3.3 日志收集

**定义**：收集系统运行日志

**适用场景**：自有系统

**优缺点**：
- 优点：实时、数据量大
- 缺点：数据格式复杂、需要清洗

**示例**：用户行为日志

### 3.3.4 其他方式

| 方式 | 定义 | 适用场景 |
| :--- | :--- | :--- |
| **传感器** | 物理设备采集数据 | 物联网场景 |
| **表单录入** | 人工填写表单 | 用户反馈 |

## 3.4 数据采集的合法性与稳定性

### 3.4.1 合法性考量

- **网站的robots.txt协议**：遵守网站的爬虫协议
- **用户数据的隐私保护**：不采集敏感个人信息
- **数据使用的合规性**：确保数据使用符合法律法规

### 3.4.2 稳定性考量

- **数据源的可用性**：确保数据源稳定可用
- **采集频率的合理性**：避免过于频繁的采集
- **异常处理机制**：设计备用方案应对异常

### 3.4.3 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
| :--- | :--- | :--- |
| 数据源中断 | 网站改版、接口变更 | 监控告警、备用方案 |
| 数据格式变更 | 网站更新 | 格式自适应、版本管理 |
| 反爬机制 | IP封锁、验证码 | 代理池、限速采集 |
| 数据质量差 | 源数据错误 | 数据校验、异常过滤 |

## 3.5 课堂研讨

**讨论题**：China's Prices Project需要采集电商平台的价格数据，请设计一个数据采集方案：
